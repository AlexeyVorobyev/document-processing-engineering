# Общее описание проекта

## Цель проекта

Создание аналитической платформы для автоматизированного сбора, очистки,
предобработки, разметки и хранения документации Terraform-провайдеров с
целью обеспечения ИИ-агента актуальными и структурированными данными для
формирования корректных Terraform-конфигураций.

## Задачи команды

### Определение области проекта

**Домен:** DevOps / Infrastructure as Code (Terraform).\
**Бизнес-проблема:** ИИ-агент управления инфраструктурой нуждается в
достоверной, структурированной и версионированной документации по
Terraform-провайдерам для генерации корректных конфигураций.
Разрозненность источников, HTML-формат и различия между версиями
провайдеров затрудняют автоматическое использование документации и
повышают риск ошибок при управлении инфраструктурой.\
**Цель в рамках домена:** Создать платформу, обеспечивающую
автоматизированную подготовку документации провайдеров Terraform для
использования ИИ-агентом.

### Сбор данных

**Определение источников данных:** - официальные сайты документации
Terraform-провайдеров; - реестры провайдеров с информацией о версиях; -
страницы ресурсов, data sources, примеров и руководств.

**Сбор данных из выбранных источников:** - автоматизированный поиск и
формирование списка провайдеров и их версий; - web-scraping страниц
документации для каждой пары «провайдер -- версия»; - сохранение сырых
HTML-документов с метаданными: URL, версия провайдера, тип страницы.

### Очистка данных

**Анализ собранных данных:** - выявление неинформативных элементов
(меню, навигация, служебные блоки); - проверка полноты основного
контента.

**Очистка данных:** - удаление навигационных и вспомогательных
элементов; - извлечение основного контента страницы; - сохранение
структуры заголовков, таблиц аргументов, вложенных блоков и примеров
HCL-кода; - преобразование HTML в чистый текстовый формат.

### Предобработка данных

**Нормализация и стандартизация данных:** - приведение текста к единому
формату; - унификация названий и структуры секций.

**Обработка пропусков и структурных несоответствий:** - восстановление
отсутствующих заголовков секций при возможности; - контроль целостности
описаний аргументов.

**Разметка данных:** - выделение логических секций (Arguments,
Attributes, Import, Examples, Notes); - привязка текста к сущности
(resource, data source) и версии провайдера; - добавление метаданных:
имя сущности, тип, версия, источник.

### Хранение данных

**Выбор хранилища:** - файловое или объектное хранилище для сырых HTML и
очищенных TXT-файлов; - база данных для метаданных; - векторная база
данных для последующей индексации текстовых фрагментов.

**Настройка хранилища и загрузка данных:** - организация структуры
хранения по провайдерам и версиям; - загрузка очищенных и размеченных
данных с сохранением связей между слоями.

### Обеспечение качества данных

**Оценка качества данных с использованием метрик:** - полнота покрытия
документации по провайдерам и версиям; - целостность и отсутствие
дубликатов; - наличие ключевых секций в документации; - корректность
привязки к версиям.

**Настройка автоматизированных проверок качества данных:** - контроль
наличия обязательных метаданных; - проверка структуры очищенного
текста; - мониторинг обновлений источников и актуальности данных.

## Что не входит в скоуп проекта

### Анализ данных и визуализация

Проект не предполагает проведение аналитического анализа данных для
выявления бизнес-инсайтов и закономерностей, а также построение
визуализаций и дашбордов.\
Подготовленные данные используются не для интерпретации человеком, а как
источник знаний для ИИ-агента.

### Подготовка отчетов с рекомендациями по улучшению бизнес-процессов

В рамках проекта не формируются аналитические отчеты и рекомендации для
принятия управленческих решений.\
Результатом является инфраструктура данных, а не выводы о состоянии или
оптимизации бизнес-процессов.

### Классическое моделирование и статистический анализ

Проект не включает построение статистических моделей,\
прогнозирование, кластеризацию или другие методы классического data
science,\
так как цель не состоит в анализе числовых показателей.

### Кодирование категориальных признаков и работа с выбросами в табличных данных

Проект не предполагает работу с табличными наборами данных и числовыми
признаками,\
поэтому задачи кодирования категориальных переменных,\
поиска и обработки выбросов и пропусков в числовых данных не
рассматриваются.
